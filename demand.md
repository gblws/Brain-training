最小MVP版本
1. 基础架构与通用需求 (3项)
- [R1] 跨端环境搭建：基于 Uni-app + Vue3 + Vite 搭建前端，确保 H5 和小程序端能正常启动。
- [R2] 后端 RESTful 骨架：基于 Spring Boot 搭建服务，实现统一结果封装类（Result）和跨域配置（CORS）。
- [R3] 通讯协议封装：前端封装 uni.request 拦截器，定义与后端约定的 JSON 数据传输标准。
2. 核心训练游戏模块 (3项)
- [R4] 舒尔特方格 (Attention)：逻辑：5x5 矩阵随机乱序、顺序点击算法、计时器精度控制。
- [R5] 斯特鲁普挑战 (Inhibition)：逻辑：字义与颜色随机匹配（一致/不一致）、计分算法。
- [R6] 记忆矩阵 (Memory)：逻辑：方格闪烁记忆、阶梯难度算法（Level-Up/Down）、图形复现校验。
3. 用户与数据中心 (4项)
- [R7] 导航系统：实现底部 TabBar（首页/我的）及页面路由切换。
- [R8] 首页 UI：游戏卡片展示及每个游戏的玩法说明（Instructions）弹窗。
- [R9] 个人中心：用户基本信息（头像、昵称、注册天数）的静态展示。
- [R10] 历史记录：从后端 GET /api/v1/user/history 接口获取 JSON 并渲染成滚动列表。
4. 数据闭环与反馈 (2项)
- [R11] 结果提交：游戏结束后自动调用 POST /api/v1/game/submit 上传得分、耗时、正确率等数据。
- [R12] 结算反馈：实时展示本次成绩。

上线完整版
1. 深度科学评估体系 (Scientific Assessment)
- [R13] 认知基准测试：用户首次登录需进行一次综合测评，确定初始能力水位。
- [R14] 大脑分值系统 (BPI)：模仿 Lumosity，将得分转化为标准化的“大脑指数”，包含不同维度（少则三个，多则五个）。
- [R15] 动态难度算法 (Adaptive Leveling)：游戏难度不再是固定的，而是根据用户能力值自动调节（IRT 理论）。
2. 用户成长与激励体系 (User Growth)
- [R17] 成就与勋章系统：例如“连续打卡 7 天”、“记忆力突破 1000 分”等可视化奖励。
- [R18] 等级提升：可提供金币奖励，金币可换取会员时间。
3. 数据分析与 AI 建议 (AI & Big Data)
- [R19] 深度训练趋势图：展示周/月/年的能力波动曲线。
- [R20] AI 智能训练导师：接入大模型，根据数据生成周报：“你的反应力超过了 80% 的同龄人，但短时记忆有所下滑，建议本周加强矩阵练习”。
4. 社交与竞技模块 (Social & Competition)
- [R22] 排行榜：基于 Redis 实现的实时排名系统。
- [R23] 1v1 实时对战：通过 WebSocket 实现双人同屏在线对战（如：看谁先点完舒尔特方格）。
5. 商业化与支付体系 (Monetization)
- [R25] 会员订阅制 (Premium)：分为基础版和专业版（解锁全部游戏、无限次练习、深度报告）。
- [R26] 微信/支付宝集成：完整的签约、扣费、开票流程。
6. 企业级系统增强 (Enterprise Grade)
- [R28] 数据库持久化：MySQL 存储用户数据，Redis 做高频缓存。
- [R29] 多端同步：网页端、小程序、App 端数据完全互通。
- [R30] 内容管理系统 (CMS)：后端管理后台，用于配置游戏参数、更新玩法说明、发送系统公告。
